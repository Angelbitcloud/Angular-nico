Las subconsultas son consultas SQL que se incluyen dentro de otras consultas. Son útiles cuando se necesita realizar una consulta más compleja que involucra datos de diferentes tablas o cuando se requiere realizar cálculos o filtrar datos basados en los resultados de una consulta anterior.

Update-Delete: Las subconsultas de actualización y eliminación se utilizan para actualizar o eliminar registros en una tabla basándose en los resultados de una subconsulta. Por ejemplo, se puede utilizar una subconsulta de actualización para modificar los valores de una columna en una tabla en función de los resultados de una consulta secundaria.

Exists-Not Exists: Las subconsultas "Exists" y "Not Exists" se utilizan para verificar la existencia de registros en una tabla basándose en los resultados de una subconsulta. Por ejemplo, se puede utilizar una subconsulta "Exists" para comprobar si existen registros en una tabla que cumplan ciertas condiciones antes de realizar una acción.

Any-Some-All: Las subconsultas "Any", "Some" y "All" se utilizan para comparar un valor con un conjunto de valores devueltos por una subconsulta. Por ejemplo, se puede utilizar una subconsulta "Any" para verificar si un valor coincide con al menos uno de los valores devueltos por la subconsulta.

Estas subconsultas permiten realizar consultas más complejas y avanzadas al combinar múltiples operaciones y criterios en una sola consulta. Ayudan a hacer las consultas más flexibles y poderosas al aprovechar los resultados de consultas secundarias para realizar acciones específicas o aplicar condiciones adicionales.
